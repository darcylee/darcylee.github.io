接口概述

接口协议：TCP

接口实现：SOCKET

数据格式：JSON

数据上传

一个完整的数据报文，由数据包头(Header)和数据体(Body)组成，数据体包括1个或者多个数据实体。Header节点名称为hd，Body节点名称为data

  1. 数据包头(hd)字段说明

    字段名称 数据类型 备注
    ptype string data
    sbh string 水表钢印号
    sim string 手机卡号
    pid Int 报文ID
    csq Int 当前gprs信号强度

  2. 数据实体(data)字段说明

    字段名称 数据类型 备注
    id Int 数据ID，一个32位长度的整形数据
    ft Float 正向累计流量
    rt Float 反向累计 流量
    fl Float 瞬时流量
    sp Float 瞬时流速
    pr Float 管道水压力（可选字段），没有接压力传感器时，数据报文不携带该字段
    bt Int 远传电池电量余量， 范围0～9
    al Int 报警信息，按位取信息 1
    ut Int 数据采集时间，采用UTC计时方式 2

  3. 完整的数据报文

    示例：

  4. 服务器应答报文
    服务器收到数据报文之后，验证JSON字段是否完整，数据是否正确，确认后回复应答，应答报文格式如下：

    字段名称 数据类型 备注
    reply string 指示哪种类型的应答，比如数据包的应答，这里应该是“data”
    pid Int 报文ID，同设备端发出的pid
    res string 数据发送状态，成功为“succ”，错误为“err”。 3

    示例：

    <!--数据发送成功-->
{
"reply": "data",
"pid": 1232,
"res": "succ"
}
<!--发生错误-->
{
"reply": "data",
"pid": 1232,
"res": "err"
}

参数更新

参数更新，包括数据采集时间，数据上传时间等信息

  • 设备端发送请求报文：

    报文字段说明：

    字段名称 数据类型 备注
    ptype string param
    sbh string 水表钢印号
    sim string 手机卡号
    pid Int 报文ID

    示例：

    {
    "ptype": "param",
    "sbh": "DHFTYC-001",
    "sim": "12345678901",
    "pid": 1234
}

  • 服务器端回复

    服务器根据设备端的请求报文中，sbh和sim信息去匹配服务器端参数，如果服务器端没有找到匹配的信息，则应答报文中res字段值为“err”，否则应答报文中res字段值为"succ"，并且带有参数对象param

    报文字段说明：

    字段名称 数据类型 备注
    reply string 指示哪种类型的应答，这里应该是“param”
    pid Int 报文ID，同设备端发出的pid
    res string succ or err,如果是succ，则带有后面的param字段
    param obj 参数对象

    param对象字段说明:

    字段名称 数据类型 备注
    upts4 Int 一天内数据上传的总次数 upload times
    interval Int 数据采集间隔，单位分钟
    locked Int 是否锁定机子。0：解锁； 非零：锁定机子
    sult5 Array（Int） 定时发送时间,为当天0点开始的秒数，如8:30分则为30600，Array的个数就是上传的时间点个数，最多设置12个，当应答不包含该字段或者携带但array大小为0，则认为采用间隔上传的方式上传数据。
    sms Array(string) 报警通知短信，string类型， Array的个数即为设置的短信通知个数。应答不携带该字段时，认为不需要短信通知

    示例：

    <!--参数获取成功-->
{
    "reply": "param",
    "pid": 1234,
    "res": "succ",
    "param": {
        "upts": 4,
        "interval": 10,
        "locked": 0,
        "sult": [
            30600,   <!--8:30-->
            35450,
            43200,
            54000,
            57600    <!--16:00-->
        ],
        "sms": [
            "13275678367",
            "15023423423",
            "17023412346",
            "13893458345"
        ]
    }
}
<!--参数获取失败-->
{
    "reply": "param",
    "pid": 1234,
    "res": "err",
}

固件更新

设备端定期发起固件升级请求，比如1天申请一次，或者指定几天申请一次，由设备端主动发起。

  • 更新流程图

  • 设备端的请求报文

    报文字段说明：

    字段名称 数据类型 备注
    ptype string update
    sbh string 水表钢印号
    sim string 手机卡号
    pid Int 报文ID
    hardware string 本机当前硬件版本号，如“v1.0.5”
    software string 本机当前软件版本号，如“v1.1.2”

    版本号定义

       v    |   1.   |   0.  | 4
------------------------------
  ver   | major  | minor | little

    一般情况，版本权值大小major > minor > little。例如: v1.3.2 > v1.2.5。

    示例：

    {
    "ptype": "update",
    "sbh": "DHFTYC-001",
    "sim": "12345678901",
    "pid": 342
    "hardware": "v1.0.2"
    "software": "v2.1.5"
}

  • 服务器回复报文

    服务器收到设备端请求更新固件的报文后，根据报文中软硬件版本和服务器上的固件版本比对结果回复不同的应答报文给设备端。这里有几种情况：

    1. 服务器上没有所请求设备的设备信息（shb和sim不匹配，或者没有信息），则res字段返回“err”信息。
    2. 服务器上的固件版本低于请求报文中的版本或者和请求报文中的版本相同，则res字段返回“updated”信息。
    3. 服务器上的固件版本高于报文中的版本，则res字段返回"succ"，并且返回FTP参数以及固件存放的路径等信息。

    报文字段说明：

    字段名称 数据类型 备注
    reply string 指示哪种类型的应答，这里应该是“update”
    pid Int 报文ID，同设备端发出的pid
    res string succ或者updated或者err,如果是succ，将出现后面的ftp字段
    ftp obj 固件地址信息

    ftp对象字段说明:

    字段名称 数据类型 备注
    hostname string ftp地址
    username string 登入用户名，这个可以做成固定的
    port Int ftp端口（1～65535）
    path6 string 固件存放的路径，建议按照硬件版本不同来设置，比如“/top/firmware/hardware_version/”
    file string 固件文件名，例如：“firm_name_software_version.firm”

    示例：

    <!--有新固件-->
{
    "reply": "update",
    "pid": 342,
    "res": "succ",
    "ftp": {
        "hostname": "ftphost/name",
        "username": "device"
        "port":3400,
        "path":"/update/firmware/hw_1.0.3/"
        "file":"dhftyc_v1.3.2.firm"
    }
}
<!--没有新固件-->
{
    "reply": "update",
    "pid": 342,
    "res": "updated",
}
<!--没有配置信息-->
{
    "reply": "update",
    "pid": 342,
    "res": "err",
}
